# 프로젝트 가이드 (쉬운 버전)

> 어려운 말 빼고, 처음 보는 사람도 5분 만에 이해하도록 다시 쓴 설명서입니다.
> 더 자세한 원본은 [프로젝트-가이드.md](프로젝트-가이드.md) 를 보세요.

---

## 1. 한 문장으로

> **네이버쇼핑 셀러를 관리하는 컨설턴트가, 매주 "누구부터 챙길지"를 빨리 정하도록 도와주는 도구입니다.**

셀러들의 성적표를 넣으면 → 점수를 매기고 → 순위를 세우고 → "이 셀러부터 보세요, 이유는 이거예요" 하는 보고서를 자동으로 써줍니다.

---

## 2. 어떤 문제를 푸나요

컨설턴트 한 명이 셀러 수십~수백 명을 맡습니다.
매주 월요일, **"이 많은 셀러 중 누구한테 먼저 연락하지?"** 를 정해야 합니다.

지금까지는:
- 엑셀에 숫자만 잔뜩 있음
- 사람이 매번 눈으로 훑어보며 골라야 함
- 느리고, 사람마다 기준이 다름

이 도구는:
- 숫자를 **점수 → 순위 → 할 일** 로 바꿔줌
- "누구부터, 왜, 무엇을 해야 하는지"가 바로 나옴

---

## 3. 어떻게 만들어졌나요 (기술은 가볍게)

- **그냥 웹페이지 하나입니다.** `index.html` 파일을 더블클릭하면 바로 열려요. 설치할 것 없음.
- 파일은 3개뿐:
  - `index.html` → 화면 (보이는 부분)
  - `styles.css` → 디자인 (색·글씨·배치)
  - `app.js` → 데이터와 계산 (머리 역할)
- **데이터는 진짜가 아니라 가짜(데모용) 셀러 14명**입니다.
- 보고서는 구글의 **Gemini AI** 가 한국어로 써줍니다.
- AI를 쓰려면 'API 키'(AI 사용 암호)가 필요한데, **이 키는 코드에 저장하지 않고** 사용자 브라우저에만 잠깐 보관합니다. (보안상 안전)

> 💡 *API 키 = AI 서비스를 쓸 때 "나 이 사람 맞아요" 하고 보여주는 비밀번호 같은 것.*

---

## 4. 셀러 한 명은 어떤 정보로 되어 있나요

각 셀러는 아래 같은 정보를 가집니다:

| 항목 | 쉬운 뜻 |
| --- | --- |
| 이름 / 카테고리 | 셀러 이름과 분야 (뷰티·패션·식품 등) |
| GMV | 한 달 매출 규모 |
| 성장률 | 매출이 오르는 중인지 떨어지는 중인지 (%) |
| 셀러 헬스 | 종합 건강 점수 (0~100, 높을수록 좋음) |
| 전환율 | 들어온 사람 중 실제로 산 비율 (%) |
| 리뷰 점수 | 별점 (5점 만점) |
| 광고 ROAS | 광고비 1원 써서 매출 몇 원 나왔나 (배수) |
| 리스크 등급 | 위험도: 안정 / 주의 / 조치 |
| 추천 액션 | "이렇게 해보세요" 하는 다음 할 일 |
| 단계 | 컨설팅이 지금 어디까지 진행됐나 |

> 숫자를 왜 이렇게 정했는지는 [데이터-가정-쉬운버전.md](데이터-가정-쉬운버전.md) 에 따로 정리했어요.

## 5. 화면 6개 — 각각 뭘 보여주나요

### ① 대시보드 — "이번 주 전체 그림 한눈에"

위쪽에 큰 숫자 카드 4개:
- **관리 GMV** : 내가 맡은 셀러들의 총 매출
- **평균 전환율** : 들어온 손님이 얼마나 잘 사는지
- **AI 브리프 대기** : 지금 당장 손봐야 할 셀러 수
- **컨설팅 헬스** : 내 담당 셀러들의 평균 건강 점수

그 아래 **'셀러 기회 맵'** (점들이 흩어진 그래프):
- **오른쪽**으로 갈수록 → 건강한 셀러
- **위쪽**으로 갈수록 → 성장 중인 셀러
- **점이 클수록** → 매출이 큰 셀러
- **점 색깔** → 🟢안정 🟠주의 🔴조치

> 보는 법: **왼쪽 아래 빨간 점 = 작고 아픈 셀러 = 제일 급함.** 점을 누르면 그 셀러 상세로 갑니다.

또 아래에는:
- **다음 추천 액션** : 챙겨야 할 셀러 상위 4명 자동 정렬 (아픈 셀러 먼저)
- **카테고리별 매출 막대** : 어느 분야가 매출을 끌고 있나
- **컨설팅 단계 현황** : 일이 어느 단계에서 막혀 있나

### ② 셀러 — "한 명 자세히 보기"

- 왼쪽에 셀러 목록 (검색·필터 가능)
- 셀러를 고르면 오른쪽에 그 사람의 모든 지표 + 추천 액션이 뜸
- 대시보드에서 "누구"를 골랐으면, 여기서 "왜"를 확인하는 곳

### ③ AI 리포트 — "주간 보고서 자동 생성" ⭐ (이 프로젝트의 핵심)

보고서 만드는 방법이 **두 가지**:

- **(A) 기본 방식 (항상 작동, 인터넷·AI 없어도 됨)**
  미리 짜둔 규칙으로 글을 조립합니다. 예를 들어:
  - 전환율이 너무 낮으면 → "손님은 오는데 안 사는 게 문제"
  - 광고 효율이 낮으면 → "광고가 손해 보는 게 문제"
  - 둘 다 괜찮으면 → "잘 크는 중, 더 키워볼 만함"

- **(B) AI 방식 (Gemini가 직접 작성)**
  'AI로 생성' 버튼을 누르면 AI가 더 자연스러운 글을 써줍니다.
  AI가 실패하거나 키가 없으면 → **자동으로 (A) 기본 방식으로 바꿔서** 항상 결과가 나옵니다. (빈 화면이 안 뜸)

> 같은 데이터로 **말투 3가지**(임원 보고용 / 컨설턴트용 / 셀러에게 보낼 용)를 골라 쓸 수 있어요.

### ④ 벤치마크 — "경쟁 도구를 알고 만들었다"

- 비슷한 도구 6개(Shopify, Amazon 등)의 장단점을 한 줄씩 비교
- "그래서 우리는 이렇게 만들자"는 제품 아이디어 3개(H1~H3)
- → 그냥 만든 게 아니라 **시장 조사를 하고 만들었다**는 증거

### ⑤ SQL 로직 — "데이터로 순위를 매기는 방법"

화면에 진짜 데이터베이스 코드(SQL)가 떠 있는데, **딱 3가지만 이해하면 됩니다:**
1. **모으기** — 각 셀러의 최근 8주 성적을 모음
2. **점수로 압축** — 그 성적들을 '기회 점수' 하나로 변환 (낮을수록 도움이 더 필요 = 점수 높음)
3. **줄 세우기** — 점수 높은 순으로 "누구부터 챙길지" 출력

> SQL 문법 자체는 안 읽어도 됩니다. 위 3줄이면 충분. (코드를 띄운 건 "SQL도 다룰 줄 안다"를 보여주려고)

### ⑥ 사용성 테스트 — "현장에서 검증하는 계획"

- 실제 컨설턴트에게 시켜볼 과제 6개 + 각각의 합격 기준
  - 예) "급한 셀러 3명 찾기 → 90초 안에"
- 성공 신호: 보고서 5분 내 작성 · 액션 수락률 60%+ 등
- → "만들고 끝"이 아니라 **숫자로 검증한다**는 태도

## 6. 솔직한 한계

- 데이터는 **다 가짜**입니다 (진짜 네이버 데이터 아님)
- AI가 쓴 글은 틀릴 수 있어서 → **사람이 고치는** 구조로 설계함
- SQL 점수 계산은 **데모용으로 단순하게** 만든 버전

**다음에 더 하고 싶은 것:**
1. 진짜 데이터 연결하기
2. AI 보고서를 컨설턴트가 고친 기록 쌓기
3. 셀러에게 보낼 말투 따로 만들기

## 7. 면접에서 이렇게 답하면 됩니다

- **"핵심 가치가 뭐죠?"**
  → "숫자를 보여주는 게 아니라, **누구부터·왜·뭘 할지**로 바꿔서 결정 시간을 줄여줍니다."
- **"왜 AI 자동화를 100% 안 했나요?"**
  → "초기엔 신뢰가 중요해서, **사람이 고칠 수 있게** 한 게 오히려 더 잘 받아들여집니다."
- **"AI는 어떻게 붙였나요?"**
  → "Gemini를 브라우저에서 호출하되 **비밀키는 코드에 안 넣고**, 실패하면 기본 템플릿으로 자동 전환됩니다."
