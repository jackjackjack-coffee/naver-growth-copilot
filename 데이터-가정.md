# 데이터 가정 — 지표별 숫자의 근거

> 이 문서는 데모 데이터의 각 지표 숫자를 **어떤 가정으로 정했는지** 설명합니다.
> 면접에서 *"왜 이 값을 썼어요?"* 라는 질문을 방어하기 위한 자료입니다.
>
> ⚠️ 모든 수치는 **합성(synthetic) 데이터**이며, 실제 NAVER 내부 수치가 아닙니다. 아래 벤치마크 범위는 *일반적인 국내 이커머스에서 통용되는 상식적 가정*이지 검증된 공식 통계가 아닙니다.

---

## 0. 왜 "작년 대비 +X%"만으로는 부족한가

흔한 설명: *"작년엔 GMV가 이랬고, 올해는 전년 대비 +10% 올랐다."*
→ 이것만으로는 **셀러를 비교하거나 우선순위를 매길 수 없습니다.** 이유:

- **같은 +10%라도 의미가 다릅니다.** 전환율 1.8%인 셀러의 +10%와 4.1%인 셀러의 +10%는 전혀 다른 상황입니다.
- **델타(변화율)는 절대 수준을 숨깁니다.** "성장 중"이어도 전환율이 바닥이면 여전히 문제 셀러입니다.
- **카테고리마다 정상 범위가 다릅니다.** 뷰티의 전환율 3%와 식품의 전환율 3%는 같은 의미가 아닙니다.

그래서 이 프로젝트는 각 지표를 **세 가지 축**으로 가정했습니다:

1. **절대 수준 벤치마크** — "좋다/나쁘다"의 기준선
2. **구간/임계값** — 도구가 실제로 판단에 쓰는 경계값
3. **카테고리 맥락** — 같은 지표라도 카테고리별로 다르게 해석

### 공통 전제: 어떤 셀러를 가정했나
데모 셀러 14곳은 **"월 GMV 4~18억 규모의 성장기 중견 셀러"** 로 가정했습니다.
→ 신규 영세 셀러(컨설팅 인력 투입 비효율)도, 초대형 브랜드(이미 전담 매니저 보유)도 아닌, **머천트 컨설팅의 핵심 대상 티어**입니다.

| 지표 | 데모 범위 | 가정한 "정상" 기준 | 도구가 쓰는 경계값 |
| --- | --- | --- | --- |
| GMV(월) | 4.3억~18.4억 | 컨설팅 대상 중견 셀러 티어 | SQL 규모 보정: 5억 초과 시 가점 |

## 2. 지표별 상세 가정

### 2-1. GMV (월 거래액) — `gmv`, 단위: 백만원
- **데모 값:** 430~1840 (= **월 4.3억~18.4억**)
- **가정 근거:** 이 구간은 "혼자 크기엔 벅차지만, 전담 매니저를 붙이기엔 애매한" 중견 셀러 티어입니다. 컨설턴트의 1:1 개입이 ROI가 나오는 현실적 대상이라고 가정했습니다.
- **도구 사용:** SQL의 규모 보정 항(`gmv_krw > 5억 → 가점`)은 "이왕이면 매출 영향이 큰 셀러를 약간 우선"하는 비즈니스 판단을 반영합니다.
- **실데이터 보정:** 실제로는 카테고리별 GMV 분위수로 티어를 나누고, 절대값 대신 **백분위**로 비교해야 공정합니다.

### 2-2. 성장률 — `growth`, 단위: %
- **데모 값:** -6.1% ~ +21.4%
- **가정 근거:** 월 단위(또는 직전 4주 대비) 등락률로 가정했고, 일반적인 셀러는 **월 ±10% 내**에서 움직인다고 봤습니다. +20%대는 뜨는 셀러, -5%대는 식는 셀러.
- **"YoY만으로 부족" 포인트:** 성장률은 *모멘텀* 신호일 뿐, 그 자체로 우선순위가 되지 않습니다. 그래서 성장률은 산점도의 **세로축(맥락)** 으로만 쓰고, 우선순위 정렬은 헬스/리스크로 합니다.
- **실데이터 보정:** **YoY(계절성 제거) + 4주 모멘텀**을 함께 봐야 정확합니다. (예: 식품은 명절 시즌성이 커서 YoY 필수)

### 2-3. 셀러 헬스 — `health`, 0~100 (파생 지표)
- **데모 값:** 39~82
- **이건 측정값이 아니라 "합성 점수"입니다.** 전환율·리뷰·ROAS·성장률을 0~100으로 묶었다고 **가정**한 종합 건강도입니다.
- **가정한 구간:** **70+ 양호 · 50~69 주의 · 50 미만 위험.** (데이터에서 조치 셀러는 대부분 헬스 ≤52, 안정 셀러는 70+에 분포하도록 맞춤)
- **주의(정직한 한계):** 리스크 등급이 헬스만으로 결정되진 않습니다. 예) `프레시마켓`은 헬스 52지만 리스크 '조치' — 콜드체인 리뷰 같은 **질적 이슈**가 등급을 끌어올린 케이스로 가정했습니다.
- **실데이터 보정:** 룰 기반 가중합 또는 간단한 회귀로 헬스를 재정의하고, 컨설턴트 피드백으로 구간을 튜닝.

### 2-4. 전환율 — `conversion`, 단위: %
- **데모 값:** 1.7% ~ 4.1%
- **가정 근거:** 국내 이커머스 평균 전환율을 대략 **2.5~3.5%대**로 보고, 4%+는 우수, 2% 안팎은 누수로 가정했습니다.
- **도구 사용(중요 임계값):** 리포트 진단 로직이 **전환율 2.5% 미만 → "유입이 주문으로 안 이어짐"** 으로 자동 판단합니다. (`reportMarkdown()`의 첫 분기)
- **실데이터 보정:** 카테고리별 평균이 달라서(고관여 상품은 낮음), **카테고리 정규화** 후 임계값 적용 필요.

### 2-5. 리뷰 점수 — `review`, 5점 만점
- **데모 값:** 3.9 ~ 4.8
- **가정 근거:** 어느 정도 자리 잡은 셀러는 보통 **4.0~4.8**에 몰리고, **4.0 미만이면 경고 신호**라고 가정했습니다. 한국 소비자는 리뷰 민감도가 높아 전환에 직접 영향.
- **도구 사용:** 우선순위 점수에 15% 비중. 진단문에서 "리뷰 점수 X/5" 근거로 인용해 **신뢰도**를 보강.

### 2-6. 광고 ROAS — `adRoas`, 배수(x)
- **데모 값:** 1.4x ~ 4.8x  (ROAS = 광고매출 ÷ 광고비)
- **가정 근거:** 마진에 따라 다르지만 **손익분기를 대략 2.5x 부근**으로 가정했습니다. 2.5x 미만은 광고가 손해, 3x+는 효율적, 4.8x는 매우 우수.
- **도구 사용(중요 임계값):** 전환율이 정상인데 **ROAS 2.5x 미만 → "광고 효율 문제"** 로 진단 (두 번째 분기).
- **실데이터 보정:** 셀러별 **마진율**을 알아야 진짜 손익분기 ROAS가 나옵니다. 마진 데이터 연동이 다음 단계.

### 2-7. 리스크 등급 — `risk` (파생 등급)
- **값:** 안정(Low) / 주의(Watch) / 조치(Action)
- **가정 근거:** 헬스 점수 + 질적 신호(리뷰 테마, 광고 누수 등)를 합쳐 사람이 매긴 등급이라고 가정했습니다. 순수 자동 산출이 아니라 **"데이터 + 현장 판단"** 의 결합.
- **도구 사용:** KPI **"AI 브리프 대기"** = 리스크 '조치' 셀러 수(데모에선 5곳). 산점도 버블 색.

### 2-8. 퍼널 단계 — `stage` (운영 상태)
- **값:** 데이터 검토 → 브리프 발송 → 컨설팅 대기 → 액션 수락
- **가정 근거:** 측정 지표가 아니라 **컨설팅 진행 상태**입니다. 어느 단계에 셀러가 몰려 병목인지 보기 위한 운영 데이터.

## 3. 한 줄 면접 방어

> **"왜 이 숫자들을 썼나요?"**
> "임의로 넣은 게 아니라, **월 4~18억 규모의 컨설팅 대상 셀러**를 가정하고, 전환율 2.5%·ROAS 2.5x 같은 **현실적 개선 임계값**을 기준으로 정상/문제 구간을 설계했습니다. 단순 전년 대비 증감이 아니라 **절대 수준 + 임계값 + 카테고리 맥락**으로 봤고, 실데이터 연결 시 카테고리별 분위수로 임계값을 재보정할 계획입니다."

## 4. 가정 → 실데이터 전환 체크리스트

- [ ] 카테고리별 실제 분포로 전환율·ROAS **임계값 재설정** (절대값 → 분위수)
- [ ] 성장률을 **YoY(계절성 제거) + 4주 모멘텀** 이원화
- [ ] 셀러 마진율 연동 → ROAS 손익분기 정밀화
- [ ] `health`/`risk`를 룰 또는 회귀로 재정의하고 컨설턴트 피드백으로 튜닝
- [ ] 시즌성 큰 카테고리(식품 등)는 별도 정규화
